## Feature Set

### MVP (Launch Version)

1. **Keyword Input**
   - Single keyword or phrase
   - Auto-suggest content type based on keyword intent

2. **SERP Analysis**
   - Fetch top 10 organic results
   - Extract title, URL, meta description
   - Fetch full page content for each

3. **Word Count Analysis**
   - Calculate average, median, range
   - Recommend target based on competition

4. **Header Extraction**
   - Pull all H2s and H3s from ranking pages
   - Identify common headers (appear 3+ times)
   - Suggest "must include" vs "nice to have"

5. **Question Mining**
   - Extract from FAQ sections on pages
   - Identify question-formatted headers
   - Note: People Also Ask requires different approach

6. **Related Topics**
   - TF-IDF style extraction of important terms
   - Filter out generic words
   - Highlight terms that appear on most ranking pages

7. **Output Formats**
   - Clean web display
   - Copy as markdown
   - Copy as plain text
   - Download as PDF (nice to have)

### Version 2 Features

8. **People Also Ask Integration**
   - Scrape PAA boxes from Google
   - Goldmine for question content

9. **Content Gap Identifier**
   - "3 of 10 pages mention X, but you could go deeper"
   - Opportunities to differentiate

10. **Readability Analysis**
    - Average sentence length
    - Paragraph length
    - Reading level of ranking content

11. **SERP Feature Detection**
    - Does a featured snippet exist?
    - What format wins it? (paragraph, list, table)
    - Optimize your content for the snippet

12. **Brief History**
    - Save past briefs
    - Compare briefs over time (SERPs change)

### The Google Problem

**Challenge:** Getting SERP results without paying for an API.

**Options:**

1. **User provides URLs manually (MVP safest)**
   - "Paste the top 5 URLs ranking for your keyword"
   - Avoids scraping issues entirely
   - More friction but 100% reliable

2. **Free SERP API tiers**
   - SerpApi: 100 free searches/month
   - ValueSERP: 25 free searches/month
   - Use for demo, require user's own key for heavy use

3. **Scrape Google directly (risky)**
   - Works but Google blocks aggressively
   - Would need proxy rotation
   - Not recommended for MVP

**Recommendation:** Start with Option 1 (user pastes URLs). Add free API tier as enhancement. Frame it as "we don't limit your searches â€” you're in control."

### Backend / Processing

**API Route Flow (MVP with manual URLs):**
```
1. Receive keyword + array of competitor URLs (5-10)
2. For each URL:
   - Fetch HTML content
   - Parse with Cheerio
   - Extract: title, headings, body text, FAQ sections
   - Calculate word count
   - Extract key phrases using TF-IDF
3. Aggregate across all pages:
   - Average word count
   - Common headings (appear 3+ times)
   - Question patterns
   - Frequent terms
4. Generate structured brief
5. Return JSON for frontend rendering
```

### Content Analysis Approach

**100% free, no-API approach:**

```javascript
// Extract page data
function analyzePage(html, url) {
  const $ = cheerio.load(html);
  
  return {
    url,
    title: $('title').text(),
    wordCount: $('body').text().split(/\s+/).length,
    h2s: $('h2').map((i, el) => $(el).text()).get(),
    h3s: $('h3').map((i, el) => $(el).text()).get(),
    faqs: extractFAQs($), // Look for FAQ schema or Q&A patterns
    keyTerms: extractKeyTerms($('body').text()),
  };
}

// Find common headers across pages
function findCommonHeaders(pages) {
  const headerCounts = {};
  pages.forEach(page => {
    page.h2s.forEach(h2 => {
      const normalized = normalizeHeader(h2);
      headerCounts[normalized] = (headerCounts[normalized] || 0) + 1;
    });
  });
  // Return headers appearing on 3+ pages
  return Object.entries(headerCounts)
    .filter(([h, count]) => count >= 3)
    .sort((a, b) => b[1] - a[1]);
}
```
